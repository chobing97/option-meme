# 구현 계획: 5분봉 파이프라인 분리 + 전체 세션 확장 + 테스트 강화

- 작성일: 2026-03-20
- 상태: 확정

---

## 목표

1. **5분봉 파이프라인 분리** — 1분봉 raw data를 리샘플링하여 별도 파이프라인으로 운영
2. **정규장 전체 세션 확장** — 장 시작 후 60분 → 정규장 전체 (KR 09:00~15:30, US 09:30~16:00)
3. **시간 경과 feature 강화** — 세션 내 위치를 반영하는 feature 추가
4. **Label shift 실험** — 피크/트로프 다음 봉(+1) vs 즉시(+0) 라벨링 비교
5. **대시보드 timeframe 구분** — 1분봉/5분봉 선택 UI
6. **단위테스트 강화** — 변경 모듈 + 기존 부족한 모듈 커버리지 확보

#### 1-3. `src/collector/resampler.py` 신규 생성

기능:
- `resample_1m_to_5m(df: pd.DataFrame) -> pd.DataFrame`
  - OHLC 집계: open=first, high=max, low=min, close=last, volume=sum
  - 세션 경계 존중 (날짜별 리샘플링)
  - source 컬럼 유지
- `resample_symbol(market, symbol) -> None`
  - 1분봉 raw 로드 → 5분봉 변환 → raw-generated에 저장
- `resample_all(market) -> None`
  - 전체 심볼 리샘플링

저장 경로: `data/raw-generated/stock/5m/{market}/{symbol}/{year}.parquet`

#### 1-4. `run_pipeline.py` CLI 수정

변경 사항:
- `--timeframe` 옵션 추가 (choices: 1m, 5m, default: 1m)
- collector 단계 후 5m 선택 시 리샘플링 자동 실행
- 모든 단계에서 timeframe에 따른 경로/config 분기
- `--label-config` choices에 L3 추가

### 리뷰어(TR) 체크 포인트

**매 Step 리뷰 시 공통:**
- 버그, 로직 에러, 보안 취약점
- 프로젝트 컨벤션 준수 (네이밍, import 스타일, 로깅 패턴)
- 테스트 커버리지 충분성

**Step 1 리뷰 (T1 Foundation) 중점:**
- `settings.py` 상수명/경로 변경이 하위 모듈과 호환되는지
- `variants.py` 의 timeframe별 config 조회 인터페이스가 명확한지
- 리샘플러 OHLCV 집계 정확성

**Step 2 리뷰 (T2+T3) 중점:**
- T2(labeler)와 T3(features)의 DataFrame 컬럼 인터페이스 일치 여부
  - `minutes_from_open` 범위 (0~389 vs 0~59)
  - `date` 컬럼 형식
  - `label` 값 (0/1/2) 의미 유지
- `extract_session()` 반환값을 T3 쪽에서 올바르게 사용하는지
- shift=0/1 에 따른 라벨 위치가 feature 계산과 충돌하지 않는지

**Step 3 리뷰 (T4 Pipeline CLI) 중점:**
- `--timeframe` 옵션이 모든 stage에 올바르게 전달되는지
- timeframe별 config 조회가 정확한지
- 기존 CLI 동작 (1m, L1/L2, M1~M4) 이 깨지지 않는지

**Step 4 리뷰 (T5+T6) 중점:**
- 대시보드 데이터 로딩 경로가 실제 파이프라인 출력 경로와 일치하는지
- 전체 테스트 통과 여부
