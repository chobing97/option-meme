## 목표

기존 trading 모듈의 백테스트를 버리고, 새로운 구조로 처음부터 구현한다.

- batch_predict 결과를 입력으로 사용 (봉마다 feature 재계산 불필요)
- Strategy / Executor 분리로 백테스트 → 실전 전환 용이
- 종목 선택 가능 (옵션 데이터가 있는 US 심볼)
- 파라미터 그리드 서치로 전략 비교
- 대시보드에서 시각적 분석

---

## 사용 가능한 종목

옵션 OHLCV 데이터가 있는 **US 12개 심볼**:

```
AAPL, AMD, AMZN, GOOGL, IWM, META, MSFT, NVDA, QQQ, SPY, TQQQ, TSLA
```

데이터 구조:
- `data/raw/options/us/{SYMBOL}/contracts.parquet` — 주별 ATM 풋옵션 계약 목록
- `data/raw/options/us/{SYMBOL}/{year}.parquet` — 1분봉 옵션 OHLCV (Databento)

## 아키텍처

```
[prediction parquet]    [options OHLCV]
       ↓                      ↓
   ┌─────────────┐    ┌──────────────┐
   │  Strategy   │    │   Executor   │
   │  (매매 판단) │    │  (가격/체결)  │
   └──────┬──────┘    └──────┬───────┘
          ↓                  ↓
        ┌──────────────────────┐
        │       Engine         │
        │  (Strategy+Executor  │
        │   연결, 봉 루프)      │
        └──────────┬───────────┘
                   ↓
        ┌──────────────────────┐
        │      Analyzer        │
        │  (지표, 비교, 리포트)  │
        └──────────────────────┘
```

실전 전환 시:
- Strategy, Analyzer: 동일
- Executor: BacktestExecutor → LiveExecutor 교체
- Engine 루프: pred_df 순회 → 실시간 봉 수신 대기

## 모듈 상세 설계

### 1. `result.py` — 데이터 모델

```python
@dataclass
class Trade:
    trade_id: int
    symbol: str
    entry_time: datetime
    entry_price: float          # 옵션 매수가
    entry_strike: float
    entry_expiry: datetime
    entry_underlying: float     # 매수 시점 주가
    exit_time: datetime | None
    exit_price: float           # 옵션 매도가
    exit_underlying: float      # 매도 시점 주가
    exit_reason: str            # TROUGH_SIGNAL / TP / SL / FORCE_CLOSE
    quantity: int
    pnl: float                  # (exit_price - entry_price) * quantity
    pnl_pct: float              # pnl / entry_price
    holding_bars: int
    holding_minutes: int

@dataclass
class BarSnapshot:
    timestamp: datetime
    symbol: str
    underlying_close: float
    peak_prob: float
    trough_prob: float
    action: str                 # BUY / SELL / ""
    reason: str
    position_qty: int
    option_mark_price: float
    cash: float
    position_value: float
    equity: float
    drawdown_pct: float

@dataclass
class SimulationResult:
    trades: list[Trade]
    snapshots: list[BarSnapshot]
    config: StrategyConfig
    metadata: dict              # market, symbols, period, timeframe
```

### 2. `strategy.py` — 매매 판단

```python
@dataclass
class StrategyConfig:
    threshold: float = 0.3
    tp_pct: float = 0.10
    sl_pct: float = -0.05
    force_close_minutes: int = 120
    max_positions: int = 1
    quantity: int = 1
    option_type: str = "put"
    strike_selection: str = "atm"

class Action(Enum):
    BUY = "BUY"
    SELL = "SELL"
    HOLD = "HOLD"

@dataclass
class ActionResult:
    action: Action
    reason: str                 # PEAK_SIGNAL / TROUGH_SIGNAL / TP / SL / FORCE_CLOSE / ""

class Strategy:
    def __init__(self, config: StrategyConfig): ...

    def on_bar(self, bar: dict, position: Position | None,
               session_minutes: int) -> ActionResult:
        """
        판단 순서:

          포지션 있을 때:
            1. 장마감 N분 전 → SELL (FORCE_CLOSE)
            2. 미실현 PnL <= sl_pct → SELL (SL)
            3. 미실현 PnL >= tp_pct → SELL (TP)
            4. trough_prob >= threshold → SELL (TROUGH_SIGNAL)
            5. 그 외 → HOLD

          포지션 없을 때:
            6. peak_prob >= threshold → BUY (PEAK_SIGNAL)
            7. 그 외 → HOLD
        """
```

### 3. `executor/base.py` — 추상 인터페이스

```python
@dataclass
class OptionContract:
    symbol: str
    strike: float
    expiry: datetime
    option_type: str
    contract_id: str

@dataclass
class Position:
    contract: OptionContract
    quantity: int
    avg_entry_price: float
    current_price: float
    unrealized_pnl_pct: float

@dataclass
class FillResult:
    status: str                 # FILLED / REJECTED
    fill_price: float
    fill_time: datetime
    contract: OptionContract | None
    reject_reason: str          # NO_LIQUIDITY / INSUFFICIENT_CASH / NO_CHAIN / ""

class Executor(ABC):
    @abstractmethod
    def get_option_chain(self, symbol, option_type, timestamp) -> list[OptionContract]: ...

    @abstractmethod
    def execute_buy(self, contract, quantity, timestamp) -> FillResult: ...

    @abstractmethod
    def execute_sell(self, contract, quantity, timestamp) -> FillResult: ...

    @abstractmethod
    def get_mark_price(self, contract, timestamp) -> float: ...

    @abstractmethod
    def get_cash(self) -> float: ...

    @abstractmethod
    def get_positions(self) -> list[Position]: ...

    @abstractmethod
    def get_position(self, symbol) -> Position | None: ...
```

### 4. `executor/backtest.py` — 과거 데이터 체결

```python
class BacktestExecutor(Executor):
    """
    초기화 시:
      - symbols 목록 받아서 해당 종목의 contracts.parquet + OHLCV parquet 로드
      - capital, slippage_pct 설정

    가격 조회:
      - get_option_chain(): contracts.parquet에서 해당 시점 유효 계약 필터
      - get_mark_price(): OHLCV에서 timestamp asof 매칭 → close 반환
      - execute_buy(): ask * (1 + slippage) 체결, volume==0이면 REJECTED
      - execute_sell(): bid * (1 - slippage) 체결

    내부 관리:
      - positions 리스트
      - cash 잔고
      - 매 update_price() 호출 시 mark-to-market
    """

    def __init__(self, symbols: list[str], market: str = "us",
                 capital: float = 10_000_000, slippage_pct: float = 0.005): ...

    def load_data(self) -> None:
        """symbols의 옵션 데이터 로드. 없는 심볼은 경고 후 스킵."""
```

### 5. `engine.py` — 루프 실행

```python
class BacktestEngine:
    def __init__(self, strategy: Strategy, executor: Executor): ...

    def run(self, pred_df: pd.DataFrame, market: str,
            session_minutes: int = 390) -> SimulationResult:
        """
        pred_df: batch_predict 결과
                 (datetime, symbol, close, peak_prob, trough_prob, minutes_from_open)

        1. pred_df를 (date, symbol) 그룹으로 정렬
        2. 각 봉에 대해:
           a. executor에서 현재 포지션 조회 + mark-to-market
           b. strategy.on_bar()로 action 결정
           c. BUY → executor.execute_buy()
              SELL → executor.execute_sell()
           d. BarSnapshot 기록
           e. BUY/SELL 성공 시 Trade 생성/완료
        3. 날짜 끝에 포지션 남아있으면 → 강제청산 처리
        4. SimulationResult 반환
        """

    def run_grid(self, pred_df, market,
                 configs: list[StrategyConfig]) -> list[SimulationResult]:
        """여러 config로 순차 실행, 결과 리스트 반환."""
```

### 6. `analyzer.py` — 분석

```python
class Analyzer:
    def compute_metrics(self, result: SimulationResult) -> dict:
        """
        - 총 수익률, 연환산 수익률
        - 승률, 평균 승/패, profit factor
        - MDD, Sharpe, Sortino
        - 총 트레이드 수, 평균 보유 시간
        - 월별/요일별 수익 분포
        - exit reason 분포
        """

    def compare(self, results: list[SimulationResult]) -> pd.DataFrame:
        """
        각 결과의 config + 핵심 지표를 한 행으로 → 비교 테이블

        threshold | tp    | sl    | return | win_rate | mdd   | sharpe | trades
        0.2       | 0.05  | -0.03 | +5.2%  | 58%      | -3.1% | 1.2    | 82
        0.3       | 0.10  | -0.05 | +12.3% | 63%      | -4.7% | 1.85   | 47
        """

    def to_dataframes(self, result: SimulationResult) -> dict[str, pd.DataFrame]:
        """trades_df, equity_df, metrics_dict를 반환 → 대시보드용"""
```

## CLI

```bash
# 단일 종목 백테스트
./optionmeme backtest --market us --symbol AAPL \
    --timeframe 1m --label-config L2 --model-config M3 \
    --threshold 0.3 --tp 0.10 --sl -0.05

# 복수 종목
./optionmeme backtest --market us --symbol AAPL MSFT NVDA \
    --timeframe 5m --label-config L2 --model-config M3

# 기간 지정
./optionmeme backtest --market us --symbol SPY \
    --date-from 2025-06-01 --date-to 2026-02-27

# 그리드 서치
./optionmeme backtest-grid --market us --symbol AAPL \
    --threshold 0.2,0.3,0.4 \
    --tp 0.05,0.10,0.15 \
    --sl -0.03,-0.05,-0.10

# 옵션 데이터 있는 종목 목록 확인
./optionmeme backtest --list-symbols
```
