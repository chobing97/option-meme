## 구조적 문제 분석

### 문제 1: 과매매 (하루 11~18회)

가장 보수적 설정(threshold=0.7)에서도 하루 11회 매매.
매 매매마다 bid-ask 스프레드 + 슬리피지로 ~2% 손실.
11회 × -2% = 하루 -22% 스프레드 비용.

### 문제 2: 초단기 보유 (평균 5~7분)

PEAK 시그널 직후 TROUGH 시그널이 발생 → 옵션가가 충분히 움직이기 전에 청산.
옵션은 기초자산 대비 레버리지가 있지만, 5분 내 변동은 스프레드를 이기지 못함.

### 문제 3: 슬리피지가 수익을 잠식

매수: ask × 1.005, 매도: bid × 0.995.
ask-bid 스프레드가 이미 ~1% → 왕복 ~3% 비용.
5분 보유의 평균 수익이 이를 초과하지 못함.

## 개선 방향

현재 전략(PEAK→풋매수, TROUGH→풋매도)을 파라미터 조정만으로는 수익을 낼 수 없음.
**구조적 변경이 필요:**

### 1. 최소 보유 시간 (Minimum Holding Period)
매수 후 최소 N분(예: 30분)은 TROUGH 시그널을 무시.
→ Strategy에 `min_holding_minutes` 파라미터 추가.

### 2. 매매 쿨다운 (Cooldown)
매도 후 N분간 재매수 금지.
→ Strategy에 `cooldown_minutes` 파라미터 추가.

### 3. 하루 최대 매매 횟수 제한
하루 1~3회로 제한하여 스프레드 비용 최소화.
→ Strategy에 `max_trades_per_day` 파라미터 추가.

### 4. 시그널 필터링 강화
peak_prob과 trough_prob의 차이가 클 때만 매매.
→ `min_prob_gap` 파라미터: peak_prob - trough_prob > 0.3 일 때만 BUY.

### 5. 옵션 선택 최적화
현재 ATM Put만 사용. OTM Put은 레버리지가 더 크지만 스프레드도 큼.
ITM Put은 스프레드가 좁지만 레버리지가 낮음.
→ 다양한 strike 실험.

### 6. 보유 전략 변경
TROUGH 시그널이 아니라 시간 기반 청산 (예: 2시간 후 무조건 청산).
→ TP/SL + 시간 기반 복합 청산.

## 근본 원인 재분석

### 가설 1: 모델 시그널이 옵션 수익으로 이어지지 않음

모델은 **주가 전환점(피크/트로프)**을 예측하지만:
- 피크 감지 → 풋 매수 → 주가가 실제로 하락해야 풋 수익
- 그런데 피크에서 주가 하락폭이 풋옵션의 bid-ask 스프레드를 이기지 못함
- 특히 TSLA는 변동성이 높아 스프레드가 넓음

### 가설 2: 옵션 스프레드가 전략 수익의 상한을 결정

```
평균 왕복 비용: ~3% (ask 매수 + bid 매도 + slippage)
평균 풋옵션 변동: 5분에 ~2%, 30분에 ~5%
→ 30분 보유해도 순수익 ~2%에 불과
→ 승률 50%여도 기대수익 0에 가까움
→ 현재 승률 3~8%이면 확실한 손실
```

### 가설 3: ATM 풋 1개 전략의 한계

하루에 1개 만기의 ATM 풋만 거래 → 옵션 선택의 유연성이 없음.
주가가 큰 폭으로 하락하는 날만 수익, 나머지는 스프레드 손실.

## 다음 단계 제안

### 방향 A: 옵션이 아닌 주식으로 백테스트
- 피크/트로프 모델을 **주식 매매** (공매도 또는 인버스 ETF)에 적용
- 옵션 스프레드 비용 제거 → 전략 유효성 순수 검증
- SQQQ, TQQQ 같은 레버리지 ETF 활용

### 방향 B: 더 긴 보유 + 더 적은 매매
- 하루 1회 매매, 장 시작 시 시그널 판단, 장 끝에 청산
- 일간 레벨의 전략 (분봉 시그널을 집계하여 일일 방향 결정)

### 방향 C: 모델 threshold를 극도로 높여서 고확신 시그널만
- threshold=0.9+ 에서만 매수 → 년 10~20회 매매
- 고확신 시그널의 정확도가 높다면 소수 매매로 수익 가능

### 방향 D: 옵션 스프레드 줄이기
- 유동성 높은 옵션(SPY, QQQ)만 거래
- ITM 옵션(델타 0.7+) 사용 → 스프레드 비율 축소
